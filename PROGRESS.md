# 项目进度与规划 (Project Progress)

本文档用于记录 `Terminal Fishing Game` 的开发进度、已完成的里程碑以及未来的迭代规划。

## 🟢 当前状态 (Current Status)
**版本**: v1.1.0 (进阶打磨版)

项目已经完成了核心游戏循环的搭建，并在 v1.1.0 中引入了动态难度、连击系统、垃圾障碍物以及本地持久化高分记录。代码健壮性也通过 Type Hints 和终端尺寸保护得到了提升。

## 🎯 已完成的里程碑 (Completed Milestones)

### Phase 1: 基础设施 (Infrastructure) - `[x]`
- [x] 设计合理的模块化项目结构 (`main.py`, `game.py`, `entities/` 等)。
- [x] 初始化 `curses` 终端环境，处理颜色对与光标隐藏。

- [x] 绘制游戏基础 UI（分数、最高分、连击数、提示信息）。
- [x] 实现极小终端尺寸的安全保护（避免抛出越界异常）。

### Phase 4: 实体动画与物理 (Entities & Physics) - `[x]`

- [x] 实现 `Fish` 类，包括多向移动、水下垂直微调与越界销毁机制。

### Phase 5: 游戏机制与交互 (Gameplay) - `[x]`
- [x] 实现基于碰撞检测的咬钩判定算法。
- [x] 实现视觉强反馈的 QTE 机制（闪烁提示与倒计时进度条）。
- [x] 建立鱼群生态系统（不同稀有度、速度、颜色和积分）。
- [x] 引入连击系统 (Combo) 和负面实体 ("垃圾")，增加游戏风险收益。

- [x] 实现本地存档：通过 `save.json` 持久化保存玩家最高分。

## 🚀 未来规划 (Future Roadmap)

### v1.2.0 规划：系统解耦与拓展 (Architecture & Expansion)
- [ ] **渲染引擎抽象**：将渲染逻辑抽象为 `Renderer` 接口，以便未来脱离 `curses` 限制，支持 `blessed`、`rich` 甚至 Pygame。
- [ ] **单元测试**：编写 `pytest` 测试套件，Mock 渲染引擎以测试核心状态机流转。
- [ ] **音效系统**：如果操作系统支持，接入基于 `os` 的终端提示音系统，增强咬钩反馈。

### v1.3.0 规划：养成与图鉴 (Progression)
- [ ] **图鉴系统**：增加一个菜单界面，查看所有已解锁（钓到过）的鱼类详情。
- [ ] **商店与装备**：增加金币系统，玩家可使用金币购买更好的鱼竿（延长抛竿距离）、鱼饵（提升稀有鱼咬钩率）。
